# Интерактивная история на LangGraph с interrupt/resume

Мини-игра «выбери свою историю»: LLM генерирует завязку и варианты, пользователь выбирает в консоли, LLM дописывает концовку.

## Архитектура

```
START → story_node (LLM завязка → interrupt → resume → LLM концовка)
```

- **Состояние** (`state.py`): `TypedDict` с полями `topic`, `scene_text`, `choices`, `user_choice`, `ending`.
- **Узел** (`nodes.py`): единый узел `story_node`, который:
  1. Вызывает LLM для генерации завязки и 3 вариантов.
  2. Делает `interrupt()` с вопросом и вариантами.
  3. После `resume` получает выбор пользователя.
  4. Вызывает LLM второй раз для концовки.
  5. Возвращает обновлённое состояние.
- **Главный модуль** (`main.py`): сборка графа с `InMemorySaver`, цикл `stream()`, обработка `__interrupt__`, `questionary.select`, `Command(resume=...)`.

## Установка

```bash
pip install -r requirements.txt
```

## Настройка API-ключа

Скопируйте шаблон и вставьте свой ключ:

```bash
cp .env.example .env
# Отредактируйте .env, вставив OPENAI_API_KEY
```

Или установите переменную окружения напрямую:

```bash
export OPENAI_API_KEY="your_key_here"
```

Для совместимых провайдеров (LM Studio, Ollama) укажите `OPENAI_BASE_URL`.

## Запуск

```bash
# С дефолтной темой «космический кот»
python main.py

# С произвольной темой
python main.py "фэнтези библиотека"
python main.py "киберпанк детектив"
```

## Как это работает

1. Граф запускается с начальным состоянием (тема).
2. Узел вызывает LLM → получает завязку и 3 варианта.
3. `interrupt()` ставит выполнение на паузу.
4. Цикл `stream()` обнаруживает `__interrupt__`, показывает вопрос через `questionary.select`.
5. Пользователь выбирает вариант → ответ записывается в payload.
6. `Command(resume=payload)` возобновляет граф.
7. Узел продолжает выполнение → второй вызов LLM → концовка.
8. Итоговая история выводится в консоль.

## Примечания

- Версии `langchain==1.2.10` и `langchain-openai==1.1.9` из условия отсутствуют в PyPI; используются актуальные стабильные версии (`langchain>=0.3.0`, `langchain-openai>=0.2.0`).
- При ошибке парсинга ответа LLM используются fallback-значения для вариантов.
- `InMemorySaver` хранит чекпоинты в памяти; для персистентности можно заменить на `SqliteSaver`.
